# codegraph — параллельный навигационный слой

Локальный граф кода поверх всего репозитория: где определён символ, какая у него
сигнатура, кто его импортирует/вызывает. **Дополняет** `component-map.md` и
`system-patterns/`, не заменяет их. Инструмент: [codegraph](https://github.com/colbymchenry/codegraph)
(tree-sitter + SQLite), пин-версия `@colbymchenry/codegraph@0.9.1`.

> Этот слой ставит установщик Memory Bank. Если при установке индекса ещё не
> было — он строится; если уже был — используется существующий, не пересобирается.

## Как запустить локально (≤3 шага)

1. **Снимок графа** (коммитимый артефакт): `npm run codegraph:snapshot`
   → перестраивает индекс и пишет `memory-bank/codegraph/graph.json` + `meta.json`.
   (или напрямую: `node --disable-warning=ExperimentalWarning scripts/codegraph-snapshot.mjs`)
2. **Живой запрос агентом** (опционально): MCP-сервер прописан в `.mcp.json` в корне
   репо — перезапусти Claude Code и подтверди сервер `codegraph`; появятся tools
   `codegraph_context / search / callers / callees / explore`. npx тянет пакет сам.
3. **Без агента**: читай `graph.json` напрямую или `sqlite3 .codegraph/codegraph.db`.

Зависимости: Node ≥ 22 (для `node:sqlite` в export-скрипте) + сеть для первого
`npx` (дальше из кэша). Скрипт падает с понятным кодом возврата при проблемах
(1 = нет npx, 2 = пустой индекс, 3 = нет node:sqlite).

## Артефакты и версионирование

| Файл | Что | Свойство |
| --- | --- | --- |
| `graph.json` | `{ nodes, edges }`, отсортировано, без таймстемпов | **байт-стабилен** → воспроизводим (`md5` совпадает между прогонами на неизменённом коде) |
| `meta.json` | commit SHA, `generatedAt`, версия, counts, runtime | **volatile** — расходится только в `generatedAt` |
| `.codegraph/codegraph.db` | живой SQLite-индекс | **gitignored**, эфемерный, строится на каждой машине |

Перегенерация = повторный `npm run codegraph:snapshot` (перезапись). Привязка к
коммиту — поле `commitSha` в `meta.json`. Воспроизводимость проверяется по
`graph.json` (не по `meta.json` и не по `.db` — `.db` не байт-стабилен из-за
внутренних таймстемпов).

**Когда пересобирать:** живой индекс (`.codegraph/`) обновляется сам через
file-watcher MCP-сервера, пока редактор открыт — для запросов руками ничего не
надо. Коммитимый снимок освежается **на закрытии задачи (`/archive`)**, если
менялся код — отдельно помнить не нужно. Вне workflow — вручную перед
шарингом/релизом.

## Чему верить, а чему нет (trust boundary)

> **Граф = быстрый лид, НЕ пруф полноты.**

| Запрос | Доверие |
| --- | --- |
| Где определён символ X, сигнатура, exported? | ✅ точно |
| Кто вызывает X? | ✅ scope-aware (одноимённые функции резолвятся в локальную) |
| Кто импортирует X (по имени)? | ✅ обычно верно (даже сквозь barrel) |
| Что ещё сломается, если тронуть X (blast-radius)? | ⚠️ **проверяй в коде** — резолв импортов в файл-ноды частичный |
| Логика внутри функции/компонента (локальные `const fn = …`) | ⚠️ **не индексируется**, только модульные символы |
| «Как/зачем это работает» (смысл) | ⚠️ `context` (NL) слабоват; читай код + `component-map` |

**Правило при расхождении:** структурный факт (где/кто-вызывает) — верь графу (он
из текущего кода). Замысел/история — `component-map.md`. Если граф противоречит
`component-map` на структуре — `component-map` протух, поправь его.

## Полоса ответственности (vs component-map.md)

- `component-map.md` (руками) — **зачем / история / грабли**.
- `codegraph` (авто) — **где / кто-вызывает / сигнатуры**, всегда свежий после снимка.

Аудитории разные: `component-map` читает человек, граф запрашивает агент (MCP).

## Откат

`npm`-uninstall не нужен (npx). Удалить слой целиком: `rm -rf memory-bank/codegraph/
.codegraph .mcp.json scripts/codegraph-snapshot.mjs` + убрать npm-скрипт и
указатели. `npx @colbymchenry/codegraph@0.9.1 uninit` снесёт `.codegraph/`.
