## Когда заводить новый паттерн

Новый файл-паттерн создаётся, когда:
- Один и тот же подход применился **в 2+ местах** проекта.
- Из рефлексии (`/reflect`) выяснилось, что есть нетривиальное решение, которое стоит зафиксировать.
- Реестр уроков (`/reflect` шаг 3a) пометил повторяющийся урок `[★2+]` → кандидат на новый паттерн.

## Как назвать файл

- `kebab-case-описание-сути.md` (не `pattern1.md`, не `react-stuff.md`).
- Имя — про **что**, а не про **где**: `optimistic-mutation-cache.md`, не `useTask.md`.
- Файл должен быть **читаемым в отрыве от задачи**, в которой родился.

## Структура файла-паттерна

```markdown
# <Название паттерна>

## Когда применять
Краткий триггер: «если вижу X — применяю этот паттерн».

## Решение
Один абзац или схема — как это делается.

## Пример (из проекта)
Минимальный код-снипет или ссылка на файл `src/.../foo.tsx:42`.

## Антипаттерн
Что **не** делать и почему.

## Связанные задачи
- #NNNN (где впервые применили / закрепили)
```
