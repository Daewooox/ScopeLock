# Codex REFLECT

Краткий режим рефлексии после BUILD.

## Как использовать

```text
reflect:
```

## Что делать

### 1. Ревью реализации

Сравни с планом, посмотри git diff, оцени качество.

### 2. Сверь уроки с реестром (lessons-registry)

Прочитай `memory-bank/reflection/lessons-registry.md` — чтобы не дублировать уроки и проверить известные ловушки проекта. После написания рефлексии (шаг 3), для каждого урока: найди совпадение в реестре → есть: подними счётчик (`[×N]`/`[★N]`) + допиши #задачу; нет: добавь строку `[×1]`. Продвигай в always-loaded память (`AGENTS.md`/auto-memory) только урок `×2+` И глобально-применимый как правило поведения.

### 3. Создай `memory-bank/reflection/reflection-NNNN-[name].md`

```markdown
# Рефлексия: [Название]

## Метаданные
- Задача: #NNNN
- Уровень: L[N]

## Резюме
## Что прошло хорошо
## Сложности
## Уроки
## Улучшения процесса
```

**Если round > 1** — обязательно секция «Почему понадобились итерации»: R1→R2 что попробовали и почему не сработало, какой признак указал бы на правильный путь раньше, что добавить в чеклист.

### 4. Обнови документацию (через significance gate)

Уровни:
- **Обзор** (`projectbrief.md`, `productContext.md`, `techContext.md`) — по умолчанию НЕ трогать. Это _текущее состояние_, не журнал.
- **Детали** (`docs/component-map.md`, `system-patterns/`, `style-guide/`) — обновляй по факту.

#### Когда трогать обзор

- **`projectbrief.md`**: новый раздел/страница/маршрут, смена стека (новая ключевая либа), новый перечислимый элемент (новый view-режим, новый тип сущности).
- **`productContext.md`**: новый user-visible flow/опция/модалка, смена default-поведения, новая граница прав, новый продуктовый принцип.
- **`techContext.md`**: новая зависимость / major-bump, новый паттерн (state/realtime/RLS/Edge/cron), новая инфра-единица (bucket/edge function/cron/env var/таблица-домен), смена deploy.

#### Когда НЕ трогать обзор

CSS-фиксы, i18n-ключи, refactor одного хука, перестановка кнопок, regression-багфиксы, точечные миграции БД без нового паттерна, iteration N+1 уже описанной фичи → только `reflection-NNNN-*.md` + `archive-NNNN-*.md`.

#### Антипаттерны

- ❌ Декорации `(Месяц YYYY)` / `(#NNNN)` в заголовках разделов обзора.
- ❌ Секции «Недавние изменения» / «История» в обзоре — это `progress.md`/`archive/`.
- ❌ Хронологический порядок секций — обзор организован по доменам.
- ❌ `Write` для перезаписи целиком — только точечный `Edit`.
- ❌ Дублировать содержимое `system-patterns/*.md` в `techContext.md` — только указатель.

#### Уровень деталей

- **`docs/component-map.md`** — новые ключевые файлы → дополни таблицу (Файл → Назначение, 5-12 слов). Новый домен → новый раздел.
- **`system-patterns/`** — паттерн применился ≥2 раз или зафиксирован в рефлексии → новый файл + регистрация в `_index.md`.
- **`style-guide/`** — при новых UI-компонентах / design tokens.

### 5. Следующий шаг

`archive:`
