# Codex CREATIVE

Краткий режим дизайн-решений для L3+ задач.

## Как использовать

```text
creative: <компонент или вопрос дизайна>
```

## Что делать

Для каждого компонента, отмеченного в плане:

1. **Требования и ограничения** — что должен делать, какие ограничения, как интегрируется.
2. **2-4 варианта** — название, плюсы, минусы, сложность.
3. **Анализ и выбор** — сравнение, рекомендуемый подход, обоснование.
4. **Гайдлайны реализации** — как, на что обратить внимание, подводные камни.

## Сохрани в

`memory-bank/creative/creative-NNNN-[feature].md` (NNNN = номер задачи).

```markdown
# Дизайн-решение: [Название]

## Метаданные
- Задача: #NNNN
- Тип: [архитектура/UI/алгоритм/API]

## Требования
## Рассмотренные варианты (2-4)
## Выбранное решение
## Обоснование
## Гайдлайны реализации
```
