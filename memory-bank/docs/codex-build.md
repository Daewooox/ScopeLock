# Codex BUILD

Короткий режим реализации по плану из Memory Bank.

## Как использовать

```text
build:
```

(Без аргументов — берёт активную задачу из `memory-bank/tasks.md`.)

## Что делать

### 0. Round detect
- Если в BEGIN-блоке задачи есть `Iteration: N` — это round N+1.
- Если пользователь репортит повторение бага — спроси одной строкой: «round N+1 или новая задача?»
- Round N+1: инкрементируй `Iteration:`, добавь строку в `Round-history:`.

### 1. Реализуй по плану

- L1: точечный фикс.
- L2: следуй шагам плана, проверяй каждый.
- L3-L4: фазами, прочитав `creative-NNNN-*.md`. НЕ переходи к следующей фазе, пока текущая не закрыта.

### 2. Тесты

Запусти тесты проекта (если есть). Падают — исправь.

### 3. Обнови Memory Bank

- В `tasks.md` (свой блок): отметь шаги `[x]`, статус `BUILD завершён`.
- В `progress.md`: добавь запись (`## [Дата] — [Название]`, `- Статус: реализация завершена`).
- В `activeContext.md`: «BUILD завершён» + «следующий шаг: reflect:».

### 4. Следующий шаг

`reflect:`
