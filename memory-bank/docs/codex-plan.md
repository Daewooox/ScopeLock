# Codex PLAN

Краткий режим планирования задач L2+ для Codex.

## Как использовать

```text
plan: <описание задачи>
```

(Запускай после `van:` для L2+. Для L1 план не нужен.)

## Что делать

### 1. Изучи контекст
- `memory-bank/tasks.md` — должна быть задача с завершённым VAN.
- `docs/component-map.md` — найти релевантные файлы.
- `system-patterns/_index.md` — релевантные паттерны.

### 2. Составь план

**L2:** список файлов + последовательность шагов + критерии успеха.

**L3:** компоненты + зависимости + сложности + критерии для каждого. Отметь creative phases.

**L4:** фазы + архитектурные решения + план миграции + порядок реализации.

### 3. Запиши в `tasks.md` (свой блок)

```markdown
### План реализации
#### Шаги:
1. [ ] Шаг 1 — файлы: [...] — критерий: [...]

#### Creative phases:
- [ ] [Компонент] — [тип]

#### Риски:
- ...
```

### 4. Следующий шаг

- Если есть creative phases → `creative:`
- Если нет → `build:`
