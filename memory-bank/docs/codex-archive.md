# Codex ARCHIVE

Короткий режим финализации задачи.

## Как использовать

```text
archive:
```

## Что делать

### 1. Создай `memory-bank/archive/archive-NNNN-[name].md`

```markdown
# Архив задачи: [Название]

## Метаданные
- ID: NNNN, даты, уровень, итераций

## Резюме
## Требования
## Итерации (если round > 1 — подробно по каждому)
## Реализация
## Изменённые файлы
## Тестирование
## Уроки

## Ссылки
- Рефлексия: ...
- Creative: ... (если было)
```

### 2. Очисти tasks.md

- Один блок задачи в файле → заменить на `<!-- TASK #NNNN ARCHIVED YYYY-MM-DD: archive/archive-NNNN-[name].md -->` + строка `*Готов к новой задаче.*`.
- Несколько блоков → только свой блок заменить на маркер с датой. Чужие — не трогать.

### 3. TTL-чистка (7 дней)

В `tasks.md` найди все `<!-- TASK #... ARCHIVED YYYY-MM-DD: ... -->`. Удали те, что старше 7 дней. **Не трогай** маркеры без даты, моложе 7 дней, и активные BEGIN/END блоки.

### 4. Обнови progress.md + activeContext.md

Точечно, не перезаписывая файлы.

### 5. Уроки — без отдельного шага

Извлечение повторяющихся уроков делается инкрементально в `/reflect` (шаг 3a — сверка каждого урока с `reflection/lessons-registry.md`), не в `/archive`. Здесь ничего делать не нужно.

### 6. Git commit (спроси пользователя)

«Сделать коммит изменений этой задачи?»
