# Codex VAN

Короткий режим запуска задач для Codex на базе локального Memory Bank.

## Как использовать

```text
van: <описание задачи>
```

## Что делать

### 1. Проверь Memory Bank
- `memory-bank/progress.md`, `projectbrief.md`, `productContext.md`, `techContext.md`, `docs/component-map.md` — должны существовать.
- Кодовая задача → сначала `docs/component-map.md`, потом `progress.md`.
- Не читай `archive/`, `reflection/`, `plans/` без явного запроса.

### 2. Оцени сложность

- **L1**: точечный фикс (1 файл).
- **L2**: 2-5 файлов, понятные требования.
- **L3**: фича среднего размера, есть UX/data решения.
- **L4**: крупная система, разбивать на фазы.

### 3. Выбери стиль

- L1: сразу реализация.
- L2: короткий план → реализация.
- L3: план с рисками → реализация.
- L4: план/фазы, не прыгать в код без выравнивания.

### 4. Короткий стартовый блок

```text
VAN: READY
Task: <описание>
Memory Bank: verified
Complexity: Level <1-4>
Mode: <implement now / plan then implement / phased plan>
First read: <какие файлы прочитаю первыми>
```

### 5. Правила работы

- Сначала локальный контекст, потом редактирование.
- Не выдумывай паттерны, если в коде уже есть.
- Не трогай чужие незавершённые изменения.
- При существенных правках — кратко сообщи, что изменено и как проверено.
