# Private beta feedback

Please spend five minutes answering these questions after one protected task.
You can paste the answers into a
[pilot report](https://github.com/Daewooox/ScopeLock/issues/new?template=pilot.yml).

1. Which OS, Node.js version, repository stack, and agent did you use?
2. Did installation and `scopelock setup` finish without help?
3. How long did it take to approve the first useful task boundary?
4. Was it clear which paths the agent could change and which were blocked?
5. Did the agent attempt an out-of-scope change? What did ScopeLock report?
6. Did `task finish` and the Flight Report help you decide whether to accept
   the work?
7. For multiple agents, was the execution order understandable and correct?
8. How many times did you need to intervene manually?
9. Did ScopeLock change anything you did not explicitly approve?
10. What is the single most important improvement before you use it again?

Do not include credentials, proprietary source, prompts, raw receipts, or
unredacted command output. Use the private security-reporting instructions in
[SECURITY.md](../SECURITY.md) for a vulnerability or possible data exposure.
